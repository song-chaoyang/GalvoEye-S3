## 第二章：PCB 焊接

### 2.1 焊接前检查

1. **检查 PCB**：确认 PCB 无短路、断路，铜箔完整
2. **清点元件**：对照 BOM 清单，确认所有元件齐全
3. **阅读原理图**：熟悉电路原理和元件位置
4. **设置烙铁温度**：建议 300-320°C（有铅焊锡）或 350-380°C（无铅焊锡）

### 2.2 焊接顺序

按照以下顺序焊接，从低到高、从小到大：

#### 第一步：贴片元件（如果有）

1. **电源去耦电容（100nF）**
   - 在每个 IC 的电源引脚附近放置
   - 涂少量助焊剂
   - 用镊子放置元件
   - 一端上锡固定，再焊另一端

2. **上拉电阻（4.7kΩ）**
   - I2C SDA 和 SCL 线上各一个
   - 一端接信号线，另一端接 3.3V

3. **其他贴片电阻**
   - 运放反馈电阻（10kΩ, 20kΩ, 100kΩ）
   - 补偿电容（100pF）

#### 第二步：IC 座 / 直插 IC

4. **MCP4822（DIP-8）**
   - 插入 IC 座（推荐）或直接焊接 IC
   - 注意方向：IC 表面缺口朝向 PCB 丝印标记方向
   - 先焊对角两个引脚固定，再焊其余引脚

5. **LM324（DIP-14）**
   - 同上，注意引脚方向
   - Pin 1 位置对准 PCB 丝印标记

6. **AMS1117-3.3（SOT-223 或 TO-252）**
   - 先焊中间散热焊盘
   - 再焊两侧引脚
   - 注意输入/输出方向

#### 第三步：电解电容

7. **10μF 电解电容**
   - **注意极性！** 长脚为正极，短脚为负极
   - PCB 丝印上有 + 号标记
   - 极性接反会爆炸

#### 第四步：排针 / 排母

8. **ESP32-S3-CAM 排母**
   - 焊接双排排母，注意间距
   - 排母朝上，方便插拔 ESP32-S3-CAM

9. **外设接口排针**
   - I2C 接口（4-pin）
   - PIR 接口（3-pin）
   - MOSFET 接口（3-pin）
   - 振镜驱动板接口（根据实际接口类型）

#### 第五步：电源接口

10. **DC 电源插座**
    - 焊接 5.5x2.1mm DC 圆头插座
    - 或者焊接 USB-C 接口（如使用 USB 供电）

11. **电源指示 LED**
    - 注意 LED 极性：长脚为正极
    - 串联 1kΩ 限流电阻

### 2.3 焊接质量检查

焊接完成后，逐项检查：

- [ ] 所有焊点光亮、呈圆锥形
- [ ] 无虚焊（引脚与焊盘未融合）
- [ ] 无连锡（相邻引脚短路）
- [ ] 无冷焊（焊点灰暗、粗糙）
- [ ] IC 方向正确
- [ ] 电解电容极性正确
- [ ] LED 极性正确
- [ ] 用万用表蜂鸣档检查电源和地之间无短路

### 2.4 清洗 PCB

1. 用棉签蘸无水酒精擦拭焊点周围
2. 清除松香残留和助焊剂
3. 自然晾干或用吹风机冷风吹干
4. 再次目视检查焊点

## 第三章：模块连接

按照 `docs/WIRING.md` 中的接线说明，逐个连接模块。

### 3.1 插入 ESP32-S3-CAM

1. 确认 ESP32-S3-CAM 的引脚方向与排母匹配
2. 轻轻插入，确保所有引脚对齐
3. 不要用力过猛，避免弯曲引脚
4. 插入前确认排母无歪斜

### 3.2 连接 MCP4822

如果 MCP4822 已焊接在 PCB 上，跳过此步骤。如果是面包板方案：

1. 将 MCP4822 插入面包板（或 IC 座）
2. 按照接线表连接 SPI 信号线：
   - GPIO1 → CS
   - GPIO2 → CLK
   - GPIO3 → SDI
   - GPIO4 → LDAC
3. 连接电源：VDD → 5V，VSS → GND
4. SHDN 引脚接 VDD

### 3.3 连接 LM324

1. 将 LM324 插入面包板（或 IC 座）
2. 连接电源：Pin 4 → +5V，Pin 11 → GND
3. 连接 MCP4822 输出到 LM324 输入：
   - VOUTA → Pin 2（通过 10kΩ 电阻）
   - VOUTB → Pin 6（通过 10kΩ 电阻）
4. 连接偏置电阻（20kΩ 分压网络）
5. 连接反馈电阻（100kΩ）和补偿电容（100pF）
6. LM324 输出连接到振镜驱动板

### 3.4 连接振镜 + 驱动板

1. 将振镜固定在安装位置（使用螺丝或双面胶）
2. 连接振镜电机线到驱动板（通常为 4 芯线：X+, X-, Y+, Y-）
3. 连接驱动板信号输入：
   - LM324 Pin 1 → 驱动板 X+
   - LM324 Pin 7 → 驱动板 Y+
   - GND → 驱动板 X-/Y-
4. 连接驱动板电源：V+ → +5V，GND → GND

> **注意：** 振镜电机线必须正确连接，接反会导致振镜不工作或损坏。请参考振镜和驱动板的说明书。

### 3.5 连接 RGB 激光模组

1. 将三个激光模组固定在光学平台上
2. 使用光学镜片（二向色镜）将三束光合为一束（如需白光）
3. 连接每个激光模组：
   - VCC → MOSFET 输出端
   - GND → GND
   - MOD → 对应 PWM 引脚（GPIO5/6/7）
4. **此时不要上电！** 确认所有连接后再通电

### 3.6 连接 VL53L1X ToF 传感器

1. 将 VL53L1X 模块插入 I2C 接口排针
2. 确认接线：
   - VCC → 3.3V
   - GND → GND
   - SDA → GPIO11
   - SCL → GPIO12
3. XSHUT 引脚接 3.3V（始终使能）

### 3.7 连接 HC-SR501 PIR 传感器

1. 将 HC-SR501 插入 PIR 接口排针
2. 确认接线：
   - VCC → 5V
   - GND → GND
   - OUT → GPIO13
3. 调节灵敏度旋钮到中间位置
4. 调节延时旋钮到最小（逆时针到底）

### 3.8 连接 IRF520 MOSFET 模块

1. 将 IRF520 模块插入 MOSFET 接口排针
2. 确认接线：
   - VCC → 5V
   - GND → GND
   - SIG → GPIO14
3. 连接激光电源：
   - V+ (Vin) → 5V 电源
   - V- (Load) → 三个激光模组的 VCC
4. 在 V- 和 GND 之间并联 1N4007 保护二极管

### 3.9 插入 MicroSD 卡

1. 将 MicroSD 卡格式化为 FAT32 格式
2. 插入 ESP32-S3-CAM 的 SD 卡槽
3. 确认卡插入到位（有咔哒声）

## 第四章：电源测试

> **重要：** 在连接激光模组之前完成电源测试！

### 4.1 上电前检查

1. **目视检查**：确认所有接线正确，无短路、无裸露导线
2. **万用表检查**：
   - 测量 5V 和 GND 之间电阻，应 > 1kΩ
   - 测量 3.3V 和 GND 之间电阻，应 > 1kΩ
   - 如果电阻过小，说明有短路，不要上电！
3. **确认激光模组未连接**：先断开激光模组电源
4. **佩戴激光防护眼镜**：以防万一

### 4.2 第一步：测试 3.3V 电源

1. 连接 USB 转 TTL 下载器到 ESP32-S3-CAM（仅供电，不烧录）
2. 用万用表测量：
   - ESP32-S3-CAM 的 3.3V 引脚电压应为 3.2-3.4V
   - VL53L1X 的 VCC 引脚电压应为 3.2-3.4V
3. 如果电压正常，ESP32-S3-CAM 的红色电源 LED 应亮起

### 4.3 第二步：测试 5V 电源

1. 接通外部 5V 电源适配器
2. 用万用表测量：
   - 5V 电源轨电压应为 4.8-5.2V
   - MCP4822 的 VDD 引脚电压应为 4.8-5.2V
   - LM324 的 VCC 引脚电压应为 4.8-5.2V
3. 用手触摸各 IC，不应有异常发热

### 4.4 第三步：测试 AMS1117-3.3

1. 测量 AMS1117 输入端电压：应为 4.8-5.2V
2. 测量 AMS1117 输出端电压：应为 3.2-3.4V
3. 如果输出电压异常，检查输入电压和负载电流

### 4.5 第四步：测试信号输出

1. 给 ESP32-S3-CAM 烧录测试固件（见第五章）
2. 用万用表或示波器测量：
   - MCP4822 的 VOUTA/VOUTB 输出（应有 0-2V 变化）
   - LM324 输出（应有 -5V ~ +5V 变化）
   - PWM 引脚输出（应有 0-3.3V 方波）
3. 如果信号正常，可以连接振镜驱动板测试

### 4.6 第五步：逐步连接外设

按以下顺序逐步连接和测试：

1. 连接振镜驱动板 → 测试振镜是否偏转
2. 连接 VL53L1X → 测试 I2C 通信
3. 连接 HC-SR501 → 测试 PIR 输出
4. 连接 IRF520 → 测试 MOSFET 开关
5. **最后**连接激光模组 → 测试激光输出

## 第五章：固件烧录

### 5.1 安装 PlatformIO

#### 方法一：VS Code 插件（推荐）

1. 下载并安装 [Visual Studio Code](https://code.visualstudio.com/)
2. 打开 VS Code，点击左侧扩展图标（或按 `Ctrl+Shift+X`）
3. 搜索 `PlatformIO IDE`，点击安装
4. 等待安装完成，VS Code 会自动重启
5. 重启后左侧会出现 PlatformIO 图标（外星人头像）

#### 方法二：命令行安装

```bash
# 安装 PlatformIO Core
pip install platformio

# 验证安装
pio --version
```

### 5.2 克隆项目代码

```bash
git clone https://github.com/960208781/GalvoEye-S3.git
cd GalvoEye-S3
```

### 5.3 编译固件

```bash
# 在项目根目录执行
pio run -e esp32-s3-cam
```

编译成功后，固件文件位于 `.pio/build/esp32-s3-cam/firmware.bin`。

### 5.4 烧录固件

#### 方法一：USB 转 TTL 下载器

1. 将 USB 转 TTL 下载器连接到 ESP32-S3-CAM：
   - 下载器 TXD → ESP32 U0RXD (GPIO16)
   - 下载器 RXD → ESP32 U0TXD (GPIO17)
   - 下载器 3.3V → ESP32 3.3V
   - 下载器 GND → ESP32 GND
2. 按住 ESP32-S3-CAM 上的 BOOT 按钮
3. 按一下 RST 按钮，然后松开 BOOT 按钮
4. ESP32-S3-CAM 进入下载模式
5. 执行烧录命令：

```bash
pio run -e esp32-s3-cam -t upload
```

6. 烧录完成后，按一下 RST 按钮重启

#### 方法二：PlatformIO 上传（自动检测）

```bash
pio run -e esp32-s3-cam -t upload --upload-port /dev/ttyUSB0
```

> **Windows 用户：** 端口名通常为 `COM3`、`COM4` 等
> **Linux 用户：** 端口名通常为 `/dev/ttyUSB0`、`/dev/ttyACM0` 等
> **macOS 用户：** 端口名通常为 `/dev/cu.usbserial-xxxx`

### 5.5 查看串口日志

```bash
pio device monitor -b 115200
```

正常启动后，串口应输出类似以下信息：

```

SD card mounted
MCP4822 initialized
VL53L1X initialized (addr: 0x29)
HC-SR501 initialized (GPIO13)

```

### 5.6 常见烧录问题

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 无法检测到端口 | 驱动未安装 | 安装 CH340 或 CP2102 驱动 |
| 烧录超时 | 未进入下载模式 | 按住 BOOT 按钮再按 RST |
| 烧录后无输出 | 波特率不对 | 尝试 9600 或 460800 |
| 编译错误 | 依赖缺失 | 执行 `pio pkg install` |

## 第七章：坐标标定

坐标标定是让激光投影仪准确地将图像投射到目标位置的关键步骤。

### 7.1 准备标定工具

1. 准备一张白纸或白色墙面作为投影面
2. 将投影仪固定在稳定的位置，距离投影面 1-2 米
3. 准备标定程序

### 7.2 运行标定程序

```bash
python main.py --mode calibrate
```

### 7.3 标定步骤

标定程序会在投影面上显示一系列测试点，你需要记录这些点的实际位置。

#### 第一步：中心点标定

1. 程序会在投影面中心投射一个激光点
2. 用笔在白纸上标记这个点的位置（中心点）
3. 按 `Enter` 确认

#### 第二步：边界标定

1. 程序依次投射 4 个角点（左上、右上、右下、左下）
2. 用笔标记每个角点的位置
3. 每次标记后按 `Enter` 确认

#### 第三步：网格标定（可选，提高精度）

1. 程序投射 5x5 的网格点阵
2. 用摄像头拍摄投影面
3. 程序自动识别并匹配网格点位置
4. 计算畸变校正参数

#### 第四步：保存标定数据

标定完成后，数据自动保存到 `calibration_data/calibration_data.json`：

```json
{
  "center": [512, 384],
  "corners": [[0, 0], [1023, 0], [1023, 767], [0, 767]],
  "grid_points": [...],
  "distortion_coeffs": [...],
  "timestamp": "2026-04-19T12:00:00"
}
```

### 7.4 验证标定

标定完成后，程序会自动投射测试图形（正方形和圆形）进行验证。如果变形严重，重新运行标定：

```bash
python main.py --mode calibrate
```

## 第八章：功能测试

### 8.1 ILDA 文件播放测试

#### 准备 ILDA 文件

1. 将 ILDA 文件（.ild）复制到 MicroSD 卡根目录
2. 推荐测试文件：简单的几何图形（圆形、方形、星形）

#### 播放测试

1. 通过 WebSocket 发送播放指令：

```bash
python main.py --mode track --host 192.168.4.1
```

在程序运行后，通过键盘快捷键或 WebSocket 指令控制 ILDA 播放：

```python
# 通过 WebSocket 发送 ILDA 播放指令（Python 示例）
import asyncio, websockets, json

async def play_ilda():
    async with websockets.connect("ws://192.168.4.1:81") as ws:
        # 发送 ILDA 播放指令
        await ws.send(json.dumps({"cmd": "play_ilda", "file": "circle.ild", "speed": 30000}))

asyncio.run(play_ilda())
```

## 第九章：安全系统测试

> **重要：** 在测试安全系统时，务必佩戴激光防护眼镜！

### 9.1 第一层保护：VL53L1X ToF 测距

1. 将 VL53L1X 传感器朝向投影区域前方
2. 在距离传感器 0.5m 处放置一个障碍物（模拟人体）
3. 观察串口日志，确认测距值正确
4. 测试激光自动关闭：
   - 当距离 < 1m 时，激光应自动关闭
   - 当距离 > 1.5m 时，激光应自动恢复

> **注意：** ToF 传感器测试通过固件自动执行，无需额外的 PC 端测试脚本。可以通过串口日志（`pio device monitor -b 115200`）观察 ToF 测距值和安全触发状态。

### 9.2 第二层保护：HC-SR501 人体感应

1. 在传感器前方 3-5m 处走动
2. 观察串口日志，确认 PIR 触发
3. 测试激光自动关闭：
   - 当检测到人体移动时，激光应立即关闭
   - 人体离开后延时 5 秒恢复

> **注意：** PIR 传感器测试通过固件自动执行，无需额外的 PC 端测试脚本。可以通过串口日志观察 PIR 触发状态。

> **注意：** HC-SR501 上电后需要约 30 秒稳定时间。

### 9.3 第三层保护：MOSFET 紧急断电

1. 通过 Web 界面或 WebSocket 发送紧急停止命令
2. 确认 MOSFET 立即关断（激光电源断开）
3. 测试恢复：
   - 发送恢复命令后，确认 MOSFET 重新导通
   - **激光不会立即恢复，需要手动确认安全后才能重新开启**

> **安全恢复确认说明：** 从紧急停止等危险状态恢复时，固件要求手动确认安全。这是防止激光在无人看管时自动恢复的安全设计。恢复流程：
> 1. 确认投影区域内无人员
> 2. 通过 WebSocket 发送恢复指令或按下 Web 界面的恢复按钮
> 3. 确认安全后手动开启激光

> **注意：** MOSFET 紧急断电测试通过固件和 WebSocket 指令执行，无需额外的 PC 端测试脚本。

### 9.4 安全系统联动测试

1. 启动完整安全系统
2. 模拟以下场景：

| 场景 | 预期行为 |
|------|---------|
| 人走近投影区域 | ToF 检测到 → 激光关闭 |
| 人在投影区域移动 | PIR 检测到 → 激光关闭 |
| 人离开后 | 延时 5 秒 → 激光恢复 |
| 发送紧急停止 | MOSFET 断开 → 激光立即关闭 |
| 安全系统故障 | 默认关闭激光（故障安全） |

## 第十章：常见问题和排错指南

### 10.1 硬件问题

#### ESP32-S3-CAM 无法启动

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| 电源 LED 不亮 | 电源未接或极性反 | 检查电源接线 |
| 串口无输出 | 波特率不对 | 尝试 9600, 115200, 460800 |
| 频繁重启 | 供电不足 | 更换更大功率电源 |
| 摄像头初始化失败 | 排线未接好 | 重新插拔摄像头排线 |

#### MCP4822 无输出

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| VOUTA/B 始终为 0 | SPI 通信失败 | 检查 CS/CLK/MOSI 接线 |
| 输出值不对 | 数据格式错误 | 检查 16-bit 数据格式 |
| 输出噪声大 | 去耦电容缺失 | 在 VDD 附近加 100nF 电容 |

#### LM324 输出异常

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| 输出始终为 0 | 电源未接 | 检查 Pin 4 (VCC) 和 Pin 11 (GND) |
| 输出饱和 | 增益过大 | 调整反馈电阻 |
| 输出偏移 | 偏置电压不对 | 调整 20kΩ 分压电阻 |
| 输出振荡 | 补偿不足 | 增加 100pF 补偿电容 |

#### 振镜不偏转

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| 完全不动 | 驱动板未供电 | 检查驱动板电源 |
| 只有一个轴动 | 信号线断 | 检查 X/Y 信号线 |
| 偏转方向反 | 信号极性反 | 交换 X+/X- 或 Y+/Y- |
| 抖动 | 信号噪声 | 使用屏蔽线，增加滤波 |

#### 激光模组不亮

| 症状 | 可能原因 | 解决方法 |
|------|---------|---------|
| 完全不亮 | MOSFET 未导通 | 检查 GPIO14 和 MOSFET 接线 |
| 亮度不可调 | PWM 信号未到达 | 检查 PWM 引脚接线 |
| 只有一种颜色 | 其他激光模组未接 | 检查对应模组接线 |
| 亮度异常 | PWM 频率不对 | 调整 PWM 频率到 5kHz |

### 10.2 软件问题

#### WiFi 连接失败

```
[ERROR] WiFi connect failed, status: 255
```

- 检查 WiFi SSID 和密码是否正确
- 确认路由器支持 2.4GHz（ESP32 不支持 5GHz）
- 尝试靠近路由器

#### SD 卡无法挂载

```
[ERROR] SD card mount failed
```

- 确认 SD 卡为 FAT32 格式
- 尝试更换 SD 卡（部分卡不兼容）
- 检查 SD 卡是否插入到位

#### I2C 通信失败

```
[ERROR] VL53L1X not found at address 0x29
```

- 使用 ESP32 的 I2C 扫描功能（通过串口日志或固件调试接口扫描 I2C 地址）
- 检查 SDA/SCL 接线
- 确认上拉电阻存在（4.7kΩ）
- 检查 VL53L1X 地址是否为 0x29

#### 编译错误

```
Error: No module named 'espressif32'
```

- 执行 `pio platform install espressif32`
- 或更新 PlatformIO：`pio upgrade`

### 10.3 性能优化

| 问题 | 优化方法 |
|------|---------|
| 扫描速度慢 | 提高 SPI 时钟频率到 20 MHz |
| 图形锯齿 | 增加插值点数 |
| WiFi 延迟高 | 使用静态 IP，减少 DNS 查询 |
| 摄像头帧率低 | 降低分辨率（QQVGA: 160x120） |
| 跟踪延迟大 | 降低摄像头分辨率，优化算法 |

### 10.4 获取帮助

如果以上方法无法解决问题：

2. 在项目 GitHub Issues 中搜索类似问题
3. 提交新的 Issue，附上：
   - 串口日志
   - 硬件配置（使用的模块型号）
   - 操作系统版本
   - 复现步骤
